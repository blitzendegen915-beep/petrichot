# fact-check

AI執筆記事の「事実の断定」を洗い出し、公式確認が必要な箇所の一覧を作る。

## 手順

1. `affiliate/content/` で以下のパターンをgrep抽出(行番号付き):
   - 金額: `[0-9]+(円|ドル)`, `\$[0-9]`, `月額`, `年額`
   - プラン名: `Plus|Team|Enterprise|無料プラン|有料プラン|Pro`
   - モデル名: `GPT-[0-9o]|DALL-E|Claude *(3|4|Opus|Sonnet|Haiku)|Gemini (Advanced|Ultra|Pro|Flash)`
   - 数値制限: `[0-9]+(文字|回|枚|時間|GB|万)(まで|の制限|に制限)`
2. 「〜の場合があります」「2026年時点」等のぼかし付きは除外し、断定調のみ残す。
3. 記事ファイルごとにグループ化したMarkdownリストを作り、ユーザーに提示(SendUserFile推奨)。
4. ユーザーが確認した修正指示に従い記事を修正 → build → 記事ファイルのみadd → commit/push。

## 注意

- 自分の知識で「正しい値」に書き換えない(それも古い可能性がある)。修正には必ずユーザー確認か公式ソースを使う。
- 判断に迷う記述は削るかぼかす方向で提案する(断定を弱める方が安全)。
